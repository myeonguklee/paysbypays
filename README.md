# 올페이즈 과제

## 요구사항 요약

- 대시보드 페이지 개발(거래 내역 리스트 포함)
- 결제/가맹점 관련 데이터 시각화

## 개발 환경

- Node.js 20.x
- 패키지 매니저: npm
- Next.js 16.0.6 (App Router)
- React 19.2.0
- TypeScript 5
- Tailwind CSS 4

## 사용 라이브러리

- @tanstack/react-query: 서버 상태 관리 및 데이터 페칭
- axios: HTTP 클라이언트
- recharts: 차트 시각화
- lucide-react: 아이콘 라이브러리
- react-hook-form: 폼 관리
- zod: 스키마 검증
- date-fns: 날짜 처리
- xlsx: Excel 내보내기

## 실행 방법

```bash
npm install
npm run dev
```

## 디자인 의도 및 주요 UI/UX 포인트

### 디자인 의도

사용자들에게 신뢰감을 줄 수 있는 파란색 계열을 메인 컬러로 선정하였습니다. 대시보드에서는 간결하고 중요한 정보 전달에 초점을 맞추었습니다. 필터 및 정렬 기능은 각 거래 내역, 가맹점 현황 페이지에서 데이터를 조작할 수 있도록 구성하였습니다.

### 주요 UI/UX 포인트

1. **렌더링 전략**: 첫 페이지인 대시보드에 SSR을 적용하여 중요한 지표가 빠르게 렌더링되도록 구현하였습니다.
2. **URL 쿼리 파라미터 활용**: 거래 내역 및 가맹점의 필터, 정렬 페이지네이션 기능을 URL 상태와 연동하였습니다. 대시보드의 결제 실패율 카드 내부 링크에서 실패한 거래들을 바로 확인하거나 데이터 분석을 위한 페이지 공유가 수월할 것이라 기대됩니다.
3. **반응형 UI**: 모바일, 태블릿, 데스크톱 환경에 최적화된 반응형 레이아웃을 제공하고자 했습니다.

## 사용한 주요 기술 및 선택 이유

1. **Next.js (App Router)**: 다양한 렌더링 전략(SSR, CSR)을 선택적으로 적용할 수 있으며, 자동 코드 스플리팅 및 폰트 최적화 등 성능 최적화 기능을 제공합니다. 대시보드 페이지는 SSR을 적용하여 초기 로딩 속도를 개선하고, 클라이언트 컴포넌트는 필요한 경우에만 CSR을 활용하였습니다.

2. **Tailwind CSS**: 유틸리티 기반 CSS로 빠른 개발이 가능하며, tailwind css의 모바일 퍼스트 전략과 `sm:`, `md:`, `lg:` 등의 접두사로 반응형 디자인을 더 쉽게 구현할 수 있습니다.

3. **TanStack Query (React Query)**: 서버 상태 관리와 캐싱 전략을 효율적으로 구현하기 위해 선택하였습니다. 특히 결제 상태, 결제 타입, 가맹점 상태 등 변경이 거의 없는 매핑 데이터는 `staleTime: Infinity`로 설정하여 불필요한 API 호출을 방지하고 서버 비용을 절감하였습니다. 동적 데이터는 기본 `staleTime: 5분`을 적용하여 적절한 캐싱과 최신성의 균형을 맞추었습니다.

4. **React Hook Form + Zod**: 불필요한 리렌더링을 줄여주는 RHF와 폼 검증을 선언적으로 관리하고, 타입 안정성 높은 스키마 검증을 위해 zod를 선택하였습니다.
